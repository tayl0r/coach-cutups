# Hide-PiP Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add per-clip "show PiP" with a global default; suppress the webcam picture-in-picture overlay in preview and export when the per-clip flag is off. Recording is unchanged.

**Architecture:** A single `showPiP: Bool` (default `true`) lives on `Clip` and on `Preferences.pipForNewRecordings`. JSON schema bump to `formatVersion = 3` with an in-blob migrator preserves synthesized `Codable`. Export uses the custom `CompilationCompositor`, which gates its existing PiP-draw block on the per-clip flag. Preview uses AVFoundation's built-in compositor (custom one is stripped on macOS 26), so `ClipPreviewBuilder` controls visibility by conditionally omitting the webcam `AVMutableVideoCompositionLayerInstruction` from `videoComposition.instructions`. Webcam video and audio tracks remain in every composition unchanged.

**Tech Stack:** Swift 5.9, SwiftUI, AVFoundation, `VideoCoachCore` Swift Package, XCTest, `xcodebuild`, `xcodegen`.

**Spec:** `docs/superpowers/specs/2026-05-17-hide-pip-design.md`.

**Working branch:** `hide-pip` (already created and checked out).

**Canonical test commands:**

- Core package: `swift test --package-path apple/VideoCoachCore`
- App unit tests: `cd apple && xcodebuild -project VideoCoach.xcodeproj -scheme VideoCoach -only-testing:VideoCoachTests test`
- App full build (manual smoke prep): `apple/scripts/run.sh`

- [ ] **Step 7: Implement the migrator in `ProjectStore.swift`**

Replace `ProjectStore.read` and add a private `migrateIfNeeded`:

```swift
public static func read(from folder: URL) throws -> Project {
    let url = folder.appendingPathComponent(projectFileName)
    guard FileManager.default.fileExists(atPath: url.path) else {
        throw ProjectStoreError.missingProjectJSON
    }
    let data = try Data(contentsOf: url)
    let migratedData = try migrateIfNeeded(data)
    let decoder = JSONDecoder()
    decoder.dateDecodingStrategy = .iso8601
    let project = try decoder.decode(Project.self, from: migratedData)
    // Accept v1 (legacy), v2 (added .zoom event variant), v3 (added
    // Clip.showPiP + Preferences.pipForNewRecordings). Newer formats
    // are rejected so we don't silently misinterpret unknown future fields.
    if project.formatVersion < 1 || project.formatVersion > 3 {
        throw ProjectStoreError.unsupportedFormatVersion(project.formatVersion)
    }
    return project
}

/// Project file format migrator. Each `vN → v(N+1)` step is a small
/// in-blob patch — keeps `Project` / `Clip` / `Preferences` on
/// synthesized `Codable` so future field additions don't have to
/// remember to update a hand-rolled decoder. A forgotten migrator step
/// fails LOUD (decoder throws `.keyNotFound`); a forgotten
/// `decodeIfPresent` would fail SILENT (new field decodes as zero
/// without anyone noticing).
private static func migrateIfNeeded(_ data: Data) throws -> Data {
    guard var root = try JSONSerialization.jsonObject(
        with: data, options: []) as? [String: Any]
    else { return data }
    let version = (root["formatVersion"] as? Int) ?? 1
    // Fast path for v3+ files. `ProjectStore.read` runs on every project
    // open + Open Recent — once every project has been written as v3 the
    // re-encode below would be pure overhead.
    if version >= 3 { return data }

    if var prefs = root["preferences"] as? [String: Any] {
        if prefs["pipForNewRecordings"] == nil {
            prefs["pipForNewRecordings"] = true
            root["preferences"] = prefs
        }
    }
    if var clips = root["clips"] as? [[String: Any]] {
        for i in clips.indices where clips[i]["showPiP"] == nil {
            clips[i]["showPiP"] = true
        }
        root["clips"] = clips
    }
    root["formatVersion"] = 3

    return try JSONSerialization.data(withJSONObject: root, options: [])
}

/// Test seam — `migrateIfNeeded` is `private`. This thin re-export lets
/// `ProjectTests` drive the migrator with in-memory blobs.
internal static func _testOnly_migrate(_ data: Data) throws -> Data {
    try migrateIfNeeded(data)
}
```

```swift
func test_exportSuppressesPiPWhenClipShowPiPFalse() async throws {
    // Mirrors the existing PiP-positive tests but with showPiP=false on
    // the single clip. The bottom-right corner of the output must be
    // source pixels, not the webcam's solid dark-gray.
    let clip = Clip(
        name: "noPiP",
        tags: ["test"],
        sourceIndex: 0,
        startSourceSeconds: 0,
        recordingDuration: 2,
        recordingFilename: camURL.lastPathComponent,
        events: [
            .init(recordTime: 0, kind: .zoom(.identity)),
            .init(recordTime: 0, kind: .play(sourceTime: 0)),
        ],
        showPiP: false,
        sortIndex: 0
    )

    var project = Project(name: "hide-pip-export-e2e")
    project.clips = [clip]
    let plan = project.compilationPlan(
        for: "test",
        sourceDurations: [0: Self.sourceDuration]
    )
    XCTAssertEqual(plan.entries.count, 1)

    let sourceAsset = AVURLAsset(url: srcURL)
    let webcamAsset = AVURLAsset(url: camURL)
    let exporter = CompilationExporter()
    try await exporter.export(
        plan: plan,
        clipsByID: [clip.id: clip],
        sourceAssets: [0: sourceAsset],
        clipWebcamAssets: [clip.id: webcamAsset],
        outputURL: outURL,
        resolution: .r720,
        quality: .medium,
        sourceVolume: 1.0,
        commentaryVolume: 1.0
    )
    XCTAssertTrue(FileManager.default.fileExists(atPath: outURL.path))

    // Probe inside the PiP region (CGImage coords, x=0.91, y=0.91 — same
    // location the existing PiP-positive test uses). With showPiP=false
    // this pixel comes from the `FiducialAsset` source, which at the
    // bottom-right has a magenta fiducial (0xFF, 0x00, 0xFF). With PiP
    // drawn (the regression we're guarding against), it would instead
    // be the webcam dark-gray (0x40, 0x40, 0x40 ≈ 0.25 per channel).
    let cg = try await sampleFrame(of: outURL, atOutputTime: 0.5)
    let pipProbeRect = CGRect(x: 0.91, y: 0.91, width: 0.04, height: 0.04)
    let pipRGB = PixelSampling.averageRGB(in: cg, normalizedRect: pipProbeRect)
    print("[exporter hide-PiP] bottom-right probe: \(pipRGB) — expect non-gray source pixels")
    // If the webcam dark-gray drew, every channel would be ~0.25. Any
    // single channel substantially above that means source pixels won.
    let maxChannel = max(pipRGB.r, pipRGB.g, pipRGB.b)
    XCTAssertGreaterThan(
        maxChannel, 0.40,
        "bottom-right looks like webcam dark-gray (~0.25/ch) — PiP drew despite showPiP=false. Sample=\(pipRGB)"
    )
}
```

If `Resolution` doesn't have `.r720` (it does — see `Project.swift:17`), or the `FiducialAsset` bottom-right fiducial color isn't magenta, the assertion's max-channel threshold of 0.40 still holds: anything that's NOT the uniform 0x40 webcam gray passes. The test's contract is "the bottom-right is NOT webcam gray," which is robust to which source fiducial actually lands there.
