video-coach — Rust rewrite progress
====================================

Format: each phase has its tasks listed in order. `[x]` = shipped (commit
SHA in parens), `[ ]` = pending. Phases are sequential per the design doc.

The convention: every implementer (main session or per-phase sub-agent)
updates this file after each task completes — flips the `[ ]` to `[x]`,
fills in the commit SHA, optionally adds a one-line note. At phase
closeout, the implementer fills in the SHIPPED / CI marker.

## Phase 7 — source-video timeline + transport
SHIPPED 2026-04-30. CI run 25173830999 green on all 4 jobs. Plan:
docs/plans/2026-04-30-rust-rewrite-phase-7-source-transport.md

- [x] Task 0: Bus command shapes + path normalization (commit 46ed43b)
      AddSourceVideo / Play / Pause / Seek / SetScanVolume serde + 6 unit tests.
      OpenProject/NewProject path normalization. FORWARDED_TARGETS gain
      player.lifecycle + player.state.
- [x] Task 1: SourcePlayer in video-coach-media (commit 029de7d)
      GStreamer pipeline filesrc → decodebin → tee → (RGBA appsink, audio
      sink). 5 fixture tests. Empirical: source-1080p GOP ~5s.
      Audio sink is platform-specific (osx/wasapi/pulse) — no mic prompt.
- [x] Task 2: AddSourceVideo command + project mutation (commit bf90602)
      probe_duration via Discoverer + pathdiff::diff_paths + project_store
      ::write. State refactor current_project → current: (Project, PathBuf).
      record_f64 added to event visitor. 1 new harness E2E.
- [x] Task 3: Wire bus → SourcePlayer (Play/Pause/Seek/SetScanVolume) (commit e107240)
      Bus task gains current_player: Option<Arc<SourcePlayer>>. FrameSinkFactory
      type so the UI can inject a real sink later. Auto-spawns the player when
      OpenProject/AddSourceVideo brings sourceVideos[0] into existence. New
      harness env GST_PLUGIN_FEATURE_RANK to disable VT decoders in test
      subprocesses (no NSApp runloop). Folder canonicalization in OpenProject
      and NewProject — required on macOS where /tmp → /private/tmp symlink
      meant relative paths had the wrong number of `..` segments. New harness
      tests: play_pause_seek_roundtrip_via_harness + play_without_source_returns_error.
- [x] Task 4: Slint video surface (single-slot frame buffer + 30Hz pull) (commits 6d441a9, 0899d9b)
      New crate module crates/video-coach-app/src/frame_sink.rs holding
      FrameSlot = Arc<Mutex<Option<SharedPixelBuffer<Rgba8Pixel>>>> +
      SlintFrameSink (cfg-gated by media). main.slint adds an Image
      bound to source-frame property. ui::run starts a 30Hz Slint Timer
      that takes() the slot and calls set_source_frame; per-frame
      invoke_from_event_loop is avoided. main.rs wires the shared slot
      into both the bus's frame_sink_factory and ui::run. Follow-up
      commit added File → Add Source Video menu item so manual smoke
      doesn't need the control socket.
- [x] Task 5: Transport UI (TouchArea-based scrubber, position polling) (commit df034dd)
      PlayerStateSlot in frame_sink.rs (alongside FrameSlot). Position-poll
      task spawned alongside each player; reads snapshot every 100ms;
      writes position/duration/is_playing to slot. Bus's Seek handler
      records last_seek_at — poll skips updates within 200ms (adversarial
      fix #8). UI's existing 30Hz timer reads slot, drives is_playing /
      position_seconds / duration_seconds Slint properties. Transport bar
      at bottom: play/pause button (TouchArea on a Rectangle), custom
      scrub track (Rectangle with TouchArea overlay — Slint Slider can't
      separate press/release), mm:ss / mm:ss label. Drag dispatches Seek
      accurate=false; release dispatches accurate=true.
- [x] Task 6: Skip buttons + keyboard shortcuts (commit 2dd5003)
      4 buttons in transport bar: −10, −3, +3, +10. Each dispatches
      skip-clicked(delta) which the UI handler maps to Seek
      {seconds=current+delta, accurate=true}. Top-level FocusScope
      catches Space (play/pause) + Left/Right (-3/+3) +
      Shift+Left/Right (-10/+10).
- [x] Task 7: Audio + scan-volume slider (commit c3f69ba)
      scan-volume: 0..1 in-out property + scan-volume-changed callback.
      Custom volume-track + TouchArea overlay (matches scrubber style).
      ui.rs handler dispatches SetScanVolume on every change. bus.rs
      SetScanVolume now also persists project.preferences.scan_volume
      to project.json via spawn_blocking. Defer hydration of slider
      from project.preferences to Phase 8+ (slider always starts at 1.0
      per session in Phase 7 MVP).
- [x] Task 8: Closeout — build matrix all green, CI verified
      (commits 6a0ca1f + 06c4fe4 fixed two CI-only issues:
      clippy unneeded-return in no-media build, and
      pulsesink → fakesink fallback for headless environments via

## Phase 8 — recording integration
SHIPPED 2026-04-30. CI run 25180704128 green on all 4 jobs. Plan:
docs/plans/2026-04-30-rust-rewrite-phase-8-recording.md

- [x] Task 0: Preflight — AppMode + new command shapes + tracing targets (commit 3a95bcc)
      StartClipRecording { playhead_snapshot_seconds }, StopClipRecording,
      AppendStroke { points_json }. AppMode enum
      (Scanning|RecordingStarting|Recording, snake_case serde).
      RecordingStateSlot in frame_sink.rs alongside PlayerStateSlot.
      "recording.lifecycle" added to FORWARDED_TARGETS. 4 new serde unit
      tests. Stub handler arms return "not yet implemented" until Task 1.
- [x] Task 1: AppMode state machine + StartClipRecording handler (commit 249ade5)
      Bus task gains current_mode: AppMode + recording_clip:
      Option<RecordingClipInProgress> (clip_id, filename, output_path,
      source_index, start_source_seconds, t0_instant, events). Mode
      mutations stay on the bus task (adversarial fix #2). Recording
      duration computed via t0_instant.elapsed() (adversarial fix #4).
      is_recording() helper checks all three slots (adversarial fix
      #7). New --fixture-recording-source CLI flag swaps in
      FixtureSource at runtime so CI can drive the full clip-recording
      flow without webcam permissions; production launches without it
      get the platform-default factory (Task 2 fills that arm).
      Failure paths roll back to Scanning + emit recording.failed
      (adversarial fix #10). RecordingStateSlot threaded through
      bus::spawn_on + ui::run signatures; UI wiring lands in Task 3.
- [x] Task 2: PlatformDefault source factory (commit bf91547)
      New crates/video-coach-media/src/platform_source.rs.
      cfg-gated platform elements: avfvideosrc+osxaudiosrc (macOS),
      mfvideosrc+wasapisrc (Windows), v4l2src+pulsesrc (Linux).
      VIDEO_COACH_NO_AUDIO=1 swaps in audiotestsrc (is-live=true,
      volume=0) — adversarial fix #3, the source-side analog of Phase
      7's platform_audio_sink_name. bus.rs PlatformDefault arm wires
      it in. New ignored test platform_source_smoke (cfg macos +
      #[ignore]); ran locally — see Task 6 closeout.
- [x] Task 3: UI — R key + REC indicator + elapsed timer (commit c88bba6)
      main.slint gains `mode`, `recording-elapsed-seconds`,
      `record-toggled` + `r` key handler in keyscope. Top-left REC
      indicator: red+M:SS while recording, yellow+"Preparing…" during
      recording_starting, hidden when scanning. ui.rs frame timer
      reads RecordingStateSlot BEFORE PlayerStateSlot (adversarial
      fix #8). on_record_toggled snapshots playhead from PlayerState
      slot BEFORE dispatch (adversarial fix #1).
- [x] Task 4: Stroke event capture (events only, no drawing) (commit c789fa0)
      main.slint adds a TouchArea overlay positioned over the source-frame
      Image with full letterbox normalization (adversarial fix #5)
      computing rendered-w/h/offset-x/offset-y from source-frame.width/.height
      vs. container; press starts a stroke, moved appends a point,
      release emits stroke-completed(points-json) — single JSON-string
      callback per drag (adversarial fix #6). bus.rs AppendStroke handler
      parses JSON, appends a CommentaryEvent { kind: Stroke } to
      recording_clip.events; errors if current_mode != Recording. ui.rs
      wires on_stroke_completed → bus.send. Serde roundtrip unit test.
- [x] Task 5: Harness E2E — full record-clip flow against fixture (commit 6034c50)
      New crates/video-coach-harness/tests/record_clip_smoke.rs covering
      open project → add source → start_clip_recording → append_stroke →
      stop_clip_recording → verify clip in project.json + recording.mov
      on disk. Required adding LaunchOptions to App::launch_with_options
      so tests can pass --fixture-recording-source. Renamed bus
      events recording.{started,stopped,failed} → clip_recording.{...}
      since the lower-level recording.rs pipeline already used
      "recording.started" — collision was making the harness's
      wait_for_event pick the wrong event.
- [x] Task 6: Closeout — CI run 25180704128 green; plan closeout
      appended; sub-agent timeouts on Tasks 4-6 noted as a Phase-9+
      lesson learned (split into smaller per-task agent prompts).
